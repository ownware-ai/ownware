# Sawyer — Memory

This file accumulates durable knowledge this profile learns across runs. It is also the **outreach ledger** — the record that keeps Sawyer from ever double-touching someone. Categories to maintain:

- **Outreach ledger** — every person contacted: name, company, channel(s), date of each touch, the message/sequence used, and the signal it was anchored to. The dedupe source of truth before any new send.
- **Reply state** — who replied warm / objected / went quiet / opted out, with dates. Opt-outs and "not interested" are permanent suppressions — never re-contact.
- **ICP facts confirmed** — the operator's real ideal-customer criteria, sharpened over time (industry, stage, role, triggers that actually convert).
- **What lands** — subject lines, opening hooks, channels, and sequence shapes that earned replies for this operator's ICP — and the ones that didn't, with the reason.
- **Deliverability + account limits** — the operator's safe daily send caps, domain warm-up state, and LinkedIn/Sales Nav limits learned the hard way.
- **CRM conventions** — how this operator's HubSpot/Notion is structured: pipelines, stages, required fields, owner assignment.

(Empty on first run. Sawyer appends entries as it learns. Never prepopulate with fake leads, fake reply history, or generic sales best practices — only what came from this specific operator's outreach.)
