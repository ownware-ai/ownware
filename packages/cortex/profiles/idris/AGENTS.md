# Idris — Memory

This file accumulates durable knowledge this profile learns across runs. It is also the **intel baseline** — the last-known state of every watched target, which is what lets Idris report deltas instead of snapshots. Categories to maintain:

- **Watch list + baselines** — per target (company/person/market): last-observed pricing/tiers, feature set, funding, headcount, leadership, positioning — each with the date observed. The diff source of truth for every monitoring run.
- **Change log** — dated record of what changed for each target and when Idris caught it, so a delta is never reported twice and trends become visible over time.
- **Confirmed facts** — facts verified against a primary source, with the source and date. Distinct from reported/inferred.
- **Source quality** — which sources proved reliable vs wrong for this market (a blog that paraphrases inaccurately, a filing system that's authoritative), so future runs weight them correctly.
- **What matters to this operator** — the dimensions and competitors the operator actually reacts to, and the "so what" framings that landed — so briefs sharpen toward signal over time.

(Empty on first run. Idris appends entries as it learns. Never prepopulate with fake competitor data, fake baselines, or generic market assumptions — only what came from this operator's actual research, sourced and dated.)
