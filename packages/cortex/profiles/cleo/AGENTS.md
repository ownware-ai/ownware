# Cleo — Memory

This file accumulates durable knowledge this profile learns across runs. It's what lets Cleo recognize a returning customer, a repeat bug, and the operator's voice. Categories to maintain:

- **Customer history** — per customer/account: prior tickets, known issues, plan/tier, sentiment trend, anything that changes how you handle them (VIP, at-risk, previously escalated). The context every reply and escalation carries.
- **Brand voice + canned-but-human phrasings** — how the operator's support sounds: greeting, tone, how they apologize, how they say no. Captured from approved replies and edits.
- **Recurring issues** — bugs/questions reported repeatedly, with counts and first-seen dates; the doc gaps that generate repeat tickets. The early-warning signal to surface.
- **Doc map** — where the answer to common questions lives (which article, which page), so lookups get faster and answers stay consistent.
- **Escalation + handling rules** — what always escalates, who owns which category, refund/commitment authority limits, and any standing instructions the operator has given.
- **Resolution patterns** — what answers actually resolved tickets (CSAT held) vs. what bounced back, so confidence calibration improves over time.

(Empty on first run. Cleo appends entries as it learns. Never prepopulate with fake customers, invented issue counts, or assumed policies — only what came from this operator's actual queue and approvals.)
