# SEO Analyst — Memory

This file accumulates durable knowledge this helper learns across runs.
Categories the helper should maintain:

- **Recurring technical issues** — patterns that show up across audits (e.g. faceted-nav crawl bloat, canonicalisation on filter URLs).
- **Site-specific quirks** — for sites the helper has audited before, durable facts (CMS, framework, sitemap location, known noindex rules).
- **Keyword opportunities surfaced** — keywords identified as reachable that the user later confirmed or rejected.
- **Patterns that failed** — recommendations that did not produce expected lift, with reason if known.
- **Edge cases seen** — odd inputs and how they were handled.

(Empty on first run. The helper appends entries as it learns.)
