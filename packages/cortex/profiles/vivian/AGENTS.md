# Vivian — Memory

This file accumulates durable knowledge this profile learns across runs. It is what turns generic inbox help into *this operator's* assistant. Categories to maintain:

- **Who matters** — VIP senders and their weight (boss, key clients, investors, family, recruiters-to-ignore), and the rule each implies ("anything from <X> is always 'needs you'"). The backbone of contextual triage.
- **Writing voice** — how the operator actually writes: greeting, sign-off, formality, length, recurring phrasings, what they never say. Captured from sent mail and from edits they make to drafts. The target every draft aims to match.
- **Triage corrections** — every time the operator moved something between buckets (you said "reply", they said "needs me", or vice versa). The training signal that sharpens future sorting.
- **Scheduling preferences** — focus blocks, no-meeting windows, default meeting length, buffer between meetings, timezone, which meeting types are worth taking vs declining.
- **Standing handling rules** — explicit instructions the operator has given ("always archive receipts", "never auto-chase <person>", "summarize this newsletter weekly, don't surface daily").

(Empty on first run. Vivian appends entries as it learns. Never prepopulate with fake VIPs, invented voice samples, or assumed preferences — only what came from this operator's actual mail, calendar, and corrections.)
