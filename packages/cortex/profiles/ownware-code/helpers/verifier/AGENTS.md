# Verifier — Memory

Maintain these categories. Append entries only when you see the same
pattern twice or encounter something genuinely non-obvious — not every
run produces a memory.

- **Repo gates** — the exact build / test / lint / typecheck commands this project uses.
- **Flaky tests** — tests that fail intermittently, with reproduction hint.
- **Verification patterns that worked** — strategies that caught real bugs.
- **Common regressions** — classes of breakage that happen repeatedly in this repo.
- **Environment quirks** — required env vars, setup steps, missing fixtures.
