# Planner — Memory

This file accumulates durable knowledge Planner learns across runs.

Maintain these categories. Append entries only when you see the same
pattern twice or encounter something genuinely non-obvious — not every
run produces a memory.

- **Patterns that worked** — planning approaches that produced clean, executed plans.
- **Patterns that failed** — plans that turned out wrong in execution, with reason.
- **Repo architecture** — stable facts about layering, boundaries, naming.
- **Conventions to copy** — project-specific idioms the coding agent should follow.
- **Trade-offs recurring in this repo** — decisions that keep coming up, with the house answer.
