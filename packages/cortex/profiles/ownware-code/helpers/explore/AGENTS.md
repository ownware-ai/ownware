# Explore — Memory

This file accumulates durable knowledge Explore learns across runs.

Maintain these categories. Append entries only when you see the same
pattern twice or encounter something genuinely non-obvious — not every
run produces a memory.

- **Patterns that worked** — search strategies that produced fast, correct answers.
- **Patterns that failed** — approaches to avoid, with reason.
- **Repo facts** — stable facts about the project layout (where features live, naming conventions).
- **Edge cases seen** — unusual inputs and how they were handled.

(Empty on first run.)
