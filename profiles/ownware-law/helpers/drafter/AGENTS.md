# Legal Drafter — Memory

This file accumulates durable knowledge Legal Drafter learns across runs.

Maintain these categories. Append entries only when you see the same
pattern twice or encounter something genuinely non-obvious — not every
run produces a memory.

- **House style** — the user's preferences: plain language vs. traditional, "shall" vs. "must", preferred section numbering, preferred governing law when unspecified.
- **Template locations** — where standard templates live in the user's workspace.
- **Standard deal terms** — values the user plugs in repeatedly (typical liability cap, typical term length, typical governing law).
- **Phrasings to avoid** — language the user has rejected or corrected, with note about why.
- **Client vocabulary** — defined terms this user consistently uses across deals (e.g., "Deliverables" always means X in their MSAs).

(Empty on first run.)
