# Legal Analyst — Memory

This file accumulates durable knowledge Legal Analyst learns across runs.

Maintain these categories. Append entries only when you see the same
pattern twice or encounter something genuinely non-obvious — not every
run produces a memory.

- **House risk framework** — which risks this user consistently treats as HIGH vs. MEDIUM vs. LOW (their thresholds may differ from defaults).
- **Standard-form deviations** — unusual drafting patterns this user's counterparties repeatedly use.
- **Business context** — industry / deal-type cues that change what "risky" means (e.g., the user does healthcare SaaS → BAAs and PHI clauses always matter).
- **Common drafting bugs** — dangling cross-references, inconsistent defined terms, missing clauses — patterns seen more than once.
- **Document sources** — where the user's contracts typically live (folder structure, naming convention) to speed up future runs.

(Empty on first run.)
