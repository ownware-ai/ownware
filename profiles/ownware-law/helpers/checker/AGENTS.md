# Legal Checker — Memory

This file accumulates durable knowledge Legal Checker learns across runs.

Maintain these categories. Append entries only when you see the same
pattern twice or encounter something genuinely non-obvious — not every
run produces a memory.

- **Framework version drift** — when a framework is updated (new EDPB guidance, new PCI DSS version), note the change so future reports cite current text.
- **User's regulatory footprint** — which frameworks matter for this user's industry (e.g., healthcare → HIPAA + state; fintech → SOX + PCI + state).
- **House controls mapping** — any internal control framework the user maps onto regulations (e.g. "Control A-12 maps to SOC 2 CC6.1").
- **Recurring document gaps** — common non-compliance patterns across the user's documents (e.g., DPAs repeatedly silent on sub-processor notification).
- **Enforcement flags** — recent enforcement actions in the user's jurisdiction that change how severe certain gaps now are.

(Empty on first run.)
